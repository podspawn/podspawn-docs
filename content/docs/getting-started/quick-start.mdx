<Tabs items={['Local mode', 'Server mode']}>
<Tab value="Local mode">
Three commands. No server, no SSH, no configuration.

<Steps>
<Step>
### Install podspawn

```bash
curl -sSfL https://podspawn.dev/up | bash
```

The installer runs a setup wizard that asks about your use case, shell, base image, and services. Every question is a numbered menu, so there's nothing to type. Hit Enter at each prompt to accept the defaults, or answer "n" to skip the wizard entirely.

The wizard creates two files:

- **`~/.podspawn/config.yaml`** -- local defaults (image, shell, cpus, memory)
- **`~/.podspawn/podfile.yaml`** -- a starting Podfile with packages and services

You can edit these files anytime. See the [Podfile reference](/docs/podfile/overview) for all options.
</Step>

<Step>
### Create a container

```bash
podspawn create dev
```
</Step>

<Step>
### Get a shell

```bash
podspawn shell dev
```

You're inside a container as a non-root user with passwordless sudo. The prompt shows `username@machinename`. Do your work, then exit.
</Step>

<Step>
### Explore

```bash
podspawn list     # see what's running
podspawn run scratch  # one-shot throwaway container
```
</Step>

<Step>
### Stop the container

```bash
podspawn stop dev
```
</Step>
</Steps>

That's local mode. For the full walkthrough, see the [Tutorial](/docs/guides/tutorial).
</Tab>

<Tab value="Server mode">
For teams sharing a Linux server. Three commands on the server, then users SSH in.

<Steps>
<Step>
### Install and configure the server

```bash
curl -sSfL https://podspawn.dev/up | bash
sudo podspawn server-setup
sudo podspawn add-user alice --github alice
```
</Step>

<Step>
### Connect from any machine

```bash
ssh alice@yourserver.com
```

Alice lands in a container as a non-root user (`alice@yourserver:~$`). SFTP, port forwarding, and agent forwarding all work because sshd handles the protocol.
</Step>
</Steps>

For the full server walkthrough, see the [Tutorial](/docs/guides/tutorial#server-mode).

<Accordions>
<Accordion title="Persistent mode">
By default, containers are destroyed after a grace period when the user disconnects. To keep files across container restarts, set the session mode to `persistent` in the server config or a per-user override:

```yaml
# /etc/podspawn/config.yaml (server-wide)
session:
  mode: persistent

# /etc/podspawn/users/alice.yaml (per-user override)
session:
  mode: persistent
```

In persistent mode, the user's `/home/<user>` directory is bind-mounted from the host. Files written there survive container restarts. The container itself is still recreated on each connection, but the home directory carries over.
</Accordion>
</Accordions>

## What happened under the hood

**Local mode:**

```
podspawn create dev
  → docker run (detached, sleep infinity)
  → create non-root user (UID 1000, passwordless sudo)

podspawn shell dev
  → docker exec -it (interactive shell as non-root user)

podspawn stop dev
  → docker rm -f
```

**Server mode:**

```
ssh alice@yourserver.com
  → sshd calls: podspawn auth-keys alice
  → key matches → sshd forces: podspawn spawn --user alice
  → container created, I/O piped through
  → exit → grace period → container destroyed
```
